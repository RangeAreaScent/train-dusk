# Train Dusk

> 닌텐도 DS 미학의 1-bit 픽셀아트 비주얼 노벨.
> 미스터리 + 코스믹 호러. 20분 분량 단편. 4개 엔딩.

이 문서는 클로드 코드(Claude Code)에게 프로젝트를 설명하는 가이드입니다.
이 폴더의 JSON 파일들이 게임의 데이터 베이스이고, 코드는 처음부터 작성합니다.

---

## 게임 컨셉

플레이어는 정체불명의 30대 남성이 되어 늦은 밤 지하철에서 깨어납니다. 어디로 가는 길이었는지, 자신이 누구인지조차 흐릿합니다. 칸을 하나씩 지나며 단서를 모으고, 추론하고, 결국 자신이 처한 상황의 진실에 다가갑니다.

4개의 엔딩이 있고, 모두 보기 위해서는 최소 3회차 플레이가 필요합니다. 각 회차마다 단서가 점진적으로 깊어집니다.

### 핵심 메커니즘
- **5칸 한 방향 진행**: 이전 칸으로 돌아갈 수 없음
- **회차 누적**: localStorage에 회차/엔딩/단서 누적
- **노트 추론 보드**: 모은 단서 카드를 두 개씩 연결해 통찰 생성
- **선택지 기반**: 포인트 앤 클릭 아님. 비주얼 노벨 톤
- **언어**: 한국어 원본 + 영어 각색 (메뉴에서 전환)

### 엔딩 4개
- **A — 확신 (Certainty)**: 회피 누적. 자기최면으로 위장된 평온
- **B — 동행 (Companion)**: 외로움이 만든 친구. 히든 트리거
- **C — 거울 (Mirror)**: 차장의 정체. 시점 전환
- **D — 각성 (Awakening)**: 진엔딩. 모든 퍼즐 해결 + 다른 엔딩 2개+

## 기술 스택

- **React + Vite** (단일 페이지 앱)
- **Tailwind CSS** (UI 스타일링)
- **Framer Motion** (전환 애니메이션, 페이드, 컷씬)
- **html2canvas** (엔딩 카드 PNG export)
- **localStorage** (자동 저장, 회차 메타)
- **배포**: Vercel (GitHub 연동)

### 폰트
- 한글: 단정한 손글씨체 (예: EF_Diary, 온글잎 한우리, 교보손글씨 등)
- 영문: Courier Prime (타자기 톤)

### 화면 비율
- 전체: 약 5:8 세로형 박스 + 레터박스
- 비주얼 영역: 4:3 (1-bit 픽셀아트)
- 텍스트 영역: 손글씨체 + 여백
- 버튼 행: 노트 / 설정 (2개)

## 시각 톤 가이드

### 전체 미학
**닌텐도 DS의 정돈된 미니멀리즘**. 게임보이 아님. 깔끔한 상하 분리.

### 레이아웃
```
┌─────────────────────────────────────┐
│   [상단 50%] 비주얼 4:3 1-bit 픽셀    │
│                                     │
│   ┌──────┐ ┌──────┐                 │  ← 얇은 버튼 행
│   │ 노트 │ │ 설정 │                  │
│   └──────┘ └──────┘                 │
│  ─────────────────────────────       │  ← 1px 분리선
│                                     │
│   [하단 50%] 텍스트 + 선택지          │
│   손글씨체, 여백, 깔끔               │
└─────────────────────────────────────┘
```

### 색감
- 비주얼: 순흑백 1-bit (검정/흰색만, 디더링으로 음영)
- UI: 흰 배경 + 검정 1px 라인
- 텍스트: 검정 글씨, 흰 배경
- 강조 없음, 색 강조 없음

### 버튼 클릭감
실제 물리적 누름. 클릭 시 1~2px 아래로 + 그림자 변화. CSS:
```css
.button {
  border: 1px solid #000;
  box-shadow: 0 2px 0 #000;
  transition: all 0.05s;
}
.button:active {
  transform: translateY(2px);
  box-shadow: 0 0 0 #000;
}
```

### 선택지 마커
- 일반: `▸ 가방을 연다`
- 혼잣말: `▸ "...여기가 어디지."` (살짝 회색)
- 히든: `▸ [...?]` (호버 전엔 거의 안 보임)

### 전환
- 씬 전환: 페이드 인/아웃 (0.5초)
- 컷씬: 텍스트 영역 일시 사라짐, 비주얼이 전체 화면 확장
- 아이템 팝업: 비주얼 위에 떠오름, 배경 20% 어두워짐, 위치 가변

## 텍스트 톤 원칙

### 톤의 핵심
1. **현재형** + **1인칭**
2. **짧은 문장 위주**. 한 줄씩 끊어 읽힘.
3. **묘사 < 감각**. *"의자가 있다"*보다 *"의자가 차갑다"*
4. **부정의 원칙**: 주인공은 단서를 발견할 뿐, 결론내리지 않음
   - 좋은 예: *"시간이 멈춰버린 걸까? ...그럴 리가."*
   - 나쁜 예: *"시간이 멈췄다."*
5. **`...` 자주 등장**: 침묵과 인지 사이의 시간
6. **반복이 무기**: *"괜찮아. 괜찮아."* / *"나는 그를 모른다."*

### 줄바꿈 규칙
한 문장씩, 한 줄씩. 빈 줄로 호흡. 길어지면 쉼표 대신 줄바꿈.

### 시스템 메시지
`(노트에 기록되었다: ...)` 같은 메시지는 별도 스타일 (예: 회색 이탤릭).

### 컷씬 표시
텍스트 안에 `—` 라인이 등장하면 컷씬 전환. 일러스트가 풀화면으로 표시되는 구간.

### 타이핑 효과
모든 본문 텍스트는 한 글자씩 타이핑되어 등장. 속도 설정에서 조절 가능.

## 주인공 캐릭터 시트

- **이름**: 플레이어 입력 (게임 시작 시)
- **나이**: 30대 중반
- **성별**: 남성
- **키**: 180cm, 적당한 체격
- **머리**: 장발, 갈색
- **옷차림**: 트렌치 코트 (구겨짐) + 정장 (노타이) + 가방
- **직업**: 모름 (게임 내내 단서로 추적)
- **가족**: 미혼. 어머니 + 형 + 형수
- **사고 당시 날짜**: 03.15 (게임 시작 시점)
- **진엔딩에서 드러남**: 병원 입원 중, 머리 부상, 코마/혼수 상태

## 엔딩 분기 로직

```javascript
function determineEnding(state) {
  // 최우선 — 엔딩 D
  if (
    state.insights.mustWakeUp &&
    state.endingsReached.length >= 2 &&
    state.choseAwakening
  ) return "D";

  // 엔딩 B — 친구 등장 + 손 잡음
  if (state.friendAppeared && state.choseFriendHand) return "B";

  // 엔딩 C — 차장 칸으로 향함
  if (state.choseConductorPath) return "C";

  // 기본값 — 엔딩 A
  return "A";
}
```

엔딩 D 활성화 조건:
1. 모든 단서 발견 (clues 7개 모두 true)
2. 최종 통찰 도달 (insights.mustWakeUp === true)
3. 다른 엔딩 2개 이상 도달 (endingsReached.length >= 2)
4. 칸 5에서 *"눈을 감는다"* 선택

## 노트 추론 보드 메커니즘

### 단순 인터랙션
1. 단서 카드 하나 탭/클릭 → 강조
2. 다른 단서 카드 탭/클릭 → 두 카드 사이 선이 그어짐
3. 유효 연결: 통찰 등장, 선이 실선으로 굳음
4. 무효 연결: 점선이 흩어지며 사라짐, 랜덤 무효 메시지

### 유효 연결 7개
`connections.json` 참조. 단서 6개 + 통찰 1개 (최종).

### 무효 연결 메시지 (랜덤)
한국어 5개 / 영어 5개 풀에서 랜덤. `connections.json` 참조.

### 활성화 조건
단서가 2개 이상 모이면 노트 추론 가능. 첫 회차 진입 시 안내 한 줄:
*"두 단서를 골라 이어볼 수 있다. 무언가 떠오를지도 모른다."*

## 저장/불러오기

### 자동 저장 (매 씬 진입 시)
```javascript
localStorage.setItem('trainDusk_save', JSON.stringify(gameState));
```

엔딩 도달 시 *이 엔딩 카드 공유* 옵션. URL에 `?ending=A&run=3` 같은 형태로.

### PNG export
엔딩 카드를 PNG로 저장 (html2canvas 사용).

## 에셋 (본인이 픽셀로 그릴 것)

### 배경 (1-bit 픽셀아트)
1. 지하철 칸 정면 (와이드, 칸 1)
2. 지하철 칸 측면 (탐색용)
3. 칸 사이 통로 (짧은 전환)
4. 칸 4 (공간 왜곡 — 좌석이 길게)
5. 차장 칸 (어두운 통로 끝)
6. 병실 (와이드)
7. 줌아웃 지하철 외관

### 캐릭터 표정 컷
- 주인공: 평정 / 위화감 / 공포 / 광기 웃음 / 안도 / 마주봄 (6개)
- 빅토리안 친구: 옅은 미소 / 부드러움 / 엔딩 D 미소 (3개)
- 차장: 뒷모습 / 돌아본 얼굴 (시그니처) (2개)
- 병실 인물 통합 컷: 의사/어머니/형/형수 한 장면으로 (1개)

### 컷씬 (풀화면 일러스트)
1. 엔딩 A — 사람들이 일제히 본인을 보며 웃음
2. 엔딩 B — 친구와 나란히 + 한 프레임 빈 자리
3. 엔딩 C — 차장 돌아봄 (시그니처)
4. 엔딩 C — 시점 전환, 본인이 다른 본인을 봄
5. 엔딩 D — 병실 와이드 + 친구의 미소
6. 엔딩 D — 줌아웃 (병실→지하철)

### 아이템 팝업 (작은 픽셀)
- 약통 (3변형)
- 핸드폰 잠금 화면 (2변형)
- 핸드폰 갤러리 (회차별 변형)
- 사원증 (2변형)
- 노트 (펼친 모습)
- 가방 (열린 모습)
- 꽃다발 + 카드 (3변형)
- 인터폰 (2변형)
- 비상 정지 버튼

### UI 요소
- 엔딩 카드 4종 (확신 / 동행 / 거울 / 각성)
- 제목 화면 — Train Dusk 타이틀 로고
- 노트 아이콘 (작은 픽셀)

**총합**: 약 40~50개의 1-bit 픽셀아트 에셋.

## 클로드 코드에게

이 프로젝트를 구현할 때:

1. **JSON 데이터를 game engine이 읽도록** 설계. 텍스트는 코드에 하드코딩하지 말 것.
2. **scenes.json의 흐름**을 따라 씬을 진행. 각 씬은 `text`, `choices`, `clueChecks`, `effects`를 가짐.
3. **다국어**: 모든 텍스트는 `{ko: [...], en: [...]}` 구조. `currentLang`에 따라 분기.
4. **회차별 텍스트 변형**: `variants.run2`, `variants.run3` 등이 있으면 회차에 맞춰 대체.
5. **타이핑 효과**: 본문 텍스트는 한 글자씩 등장. 선택지는 본문 완료 후 등장.
6. **돌아갈 수 없음**: 칸을 떠나면 이전 씬 비활성화.
7. **에셋이 없는 단계**: 일단 *placeholder*로 회색 박스 + 텍스트 라벨로 작업. 본인이 픽셀 작업 후 교체.

### 개발 단계
**Phase 1**: 데이터 구조 + 씬 진행 엔진 (텍스트만, 비주얼 영역 placeholder)
**Phase 2**: UI 톤 (DS 미학, 버튼 클릭감, 손글씨 폰트)
**Phase 3**: 노트 추론 보드 인터랙션
**Phase 4**: 다국어 (한/영 토글)
**Phase 5**: 저장/회차/엔딩 분기
**Phase 6**: 픽셀 에셋 통합
**Phase 7**: 컷씬 + 애니메이션 (Framer Motion)
**Phase 8**: 엔딩 카드 PNG export + URL 공유
**Phase 9**: Vercel 배포

각 Phase가 끝나면 작동하는 상태여야 함 (incremental).

이 프로젝트는 첫 게임 프로젝트입니다. 완성도 > 야심. 20분 분량의 완성된 작품을 목표로 합니다.
